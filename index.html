How It Works

  1. Storage Account mytestwebsite2025 was provisioned in the Azure Portal.
  2. Static Website Hosting was enabled — this auto-created the $web container and generated the primary endpoint URL.
  3. index.html and 404.html were uploaded into the $web container.
  4. Azure serves both files publicly over HTTPS via the dedicated endpoint.
  5. Any unknown path automatically falls back to the custom 404.html error page.

Security & Scalability

🔐 Security

  • HTTPS enforced by default on the Azure endpoint
  • Blob public access is scoped to read-only for the $web container
  • Storage account key access is managed via Azure IAM
  • No server-side code means zero runtime attack surface

📈 Scalability

  • Azure Blob Storage scales horizontally with no configuration
  • Next step: integrate Azure CDN or Azure Front Door for global edge caching and custom domain HTTPS
  • A custom domain (e.g. www.adebola.dev) would replace the default web.core.windows.net endpoint
  • Lighthouse / WebPageTest metrics would validate load-time improvements after CDN integration
